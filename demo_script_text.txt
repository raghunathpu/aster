A · PROJECT ANALYSIS
Project Summary
Field
Details
Project Name
ASTER — Adaptive Smart Traffic Event Response
Tagline
"ASTER turns incident reports into response plans — in under 2 seconds."
Problem
Bengaluru: 7M+ vehicle trips/day, 8,000+ incidents/year handled purely by officer
experience — no prediction, no standard protocol
Target Users
BTP Operations Officers · Traffic Control Room (TCR) staff · City Traffic Management
Authorities
Main Solution
ML classifier + decision engine → impact tier (Low/Med/High) + officers + barricades +
diversion + SLA + checklist, in <2 seconds
Dataset
8,173 real events from ASTRAM system (Nov 2023 – Apr 2024)
Tech Stack
Python · Gradient Boosting (sklearn) · LightGBM Cascade · Google OR-Tools MILP ·
NetworkX routing · SHAP · Streamlit · Pydeck / Leaflet
Key Accuracy
GB: 93.1% accuracy · 91.5% F1-macro · AUC-ROC 0.97 (post leakage fix — real, honest
numbers)
Setup Command
streamlit run app/aster_app.py
Key Features
Feature
What It Does
1. Dual-Model Prediction
GBM for impact tier + LightGBM cascading pipeline for resolution time, road closure
probability, and Event Impact Score (EIS)
2. SHAP Explainability
Top 3 drivers shown for every prediction in plain language — not a black box
3. Tactical Map
Leaflet map with incident pin, barricade markers, and alternative diversion routes
4. MILP Manpower
Optimizer
Google OR-Tools allocates officers across concurrent events in real time
5. Citizen Advisory
Generator
One-click WhatsApp, Twitter/X, and VMS board message drafts
6. Pre-Event Planner
Forward-looking deployment timeline for scheduled events (matches, rallies, festivals)
7. Post-Event Learning
Model drift tracker, feedback logging, one-click retraining pipeline
8. Live Feed Simulator
Sidebar button simulates live ASTRAM alert pop-ups
